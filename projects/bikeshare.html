Bikeshare NYC Analysis

Optimizing sustainable transportation in New York City through data-driven insights

Bikeshare NYC Analysis (2015-2017)

Python Data Analysis Urban Mobility Visualization

Overview

An in-depth analysis of New York City's Citi Bike system from 2015 to 2017, exploring trends and usage patterns to provide insights into urban mobility and the evolving role of bikeshare systems in city transportation.

Objectives

  • Analyze growth in ridership and usage trends
  • Identify peak usage times and popular routes
  • Explore the impact of external factors on bikeshare patterns
  • Present clear, visual insights for urban planning

Dataset Features

  • Trip duration and timing information
  • Start and end station details
  • Geographic coordinates
  • User demographics (birth year, gender)
  • User type (Customer vs Subscriber)

Key Insights

  • Analysis of trip patterns and duration
  • Popular routes and stations identification
  • User demographic analysis
  • Temporal patterns in bike usage

Technologies Used

  • Python for data analysis
  • Pandas for data manipulation
  • Matplotlib/Seaborn for visualization
  • Jupyter Notebooks for documentation

Impact

This analysis provides valuable insights into urban mobility patterns and the role of bikeshare systems in sustainable transportation. The findings contribute to the understanding of transportation patterns in major urban areas and can inform future urban planning decisions.

Data Source

Dataset sourced from Kaggle, originally from NYC Citi Bike's open data initiative.

Research Questions

The analysis focused on answering the following questions:

  1. Can the number of rides throughout the year be explored from this data set?
  2. Does the age of riders impact the amount of time spent on rides?
  3. What are the differences in environmental contributions between different user types (Customer vs. Subscriber) in the bikeshare program?
  4. Are we able to accurately predict bike share demand in different areas using our data?

Methodology

The analysis employed various data science techniques, including:

  • Exploratory data analysis to understand usage patterns
  • Visualization of temporal trends using histograms
  • Haversine formula to calculate distances between stations
  • Statistical analysis to identify popular stations and routes
# Sample code for calculating distances between stations
def haversine(lat1, lon1, lat2, lon2):
    lat1_rad = np.radians(lat1)
    lon1_rad = np.radians(lon1)
    lat2_rad = np.radians(lat2)
    lon2_rad = np.radians(lon2)

    d_lat = lat2_rad - lat1_rad
    d_lon = lon2_rad - lon1_rad
    R = 6371000

    a = np.sin(d_lat / 2)**2 + np.cos(lat1_rad) * np.cos(lat2_rad) * np.sin(d_lon / 2)**2
    c = 2 * np.arcsin(np.sqrt(a))
    distance = R * c
    return distance

bikeshare['distance(m)'] = haversine(bikeshare['Start Station Latitude'],

Key Findings

The analysis uncovered several important insights:

  • Seasonal patterns in ridership with peaks during summer months
  • Subscribers tend to take shorter, more consistent trips while Customers show higher variability in trip duration
  • The most popular station for riders to commute to/from was Grove St PATH
  • Clear patterns in demand allowed for predicting which stations would experience high traffic

Conclusions

The research provided valuable insights that can benefit the DoT in their efforts to promote eco-friendly transportation methods. By understanding demand patterns and user behavior, the Bikeshare program can be optimized to ensure proper bike availability at high-demand stations, ultimately making the service more convenient and attractive to users.

By ensuring the Bikeshare program is optimized and convenient, it will naturally gain popularity, effectively accomplishing the DoT's goal of promoting sustainable transportation.
